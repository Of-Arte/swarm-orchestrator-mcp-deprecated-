# Swarm Orchestrator v3.1: System Architecture & Release Notes

**Version**: 3.1.0 (Gemini-First Edition)
**Date**: 2026-01-20
**Architecture**: Python / FastMCP / Gemini / Docker

---

## 1. Executive Summary: The Gemini-First Migration

Swarm v3.1 represents a major architectural pivot ("Gemini-First"), moving away from fragmented model providers (OpenRouter) to a robust, unified integration with Google's Gemini ecosystem. This release consolidates all inference, embedding, and reasoning capabilities onto the Gemini API, with optional local fallbacks (Ollama) and a simplified pure-Python stack.

### Key Changes
- **Removed**: OpenRouter integration, legacy Node.js search microservice.
- **Added**: Native Gemini Embeddings (`models/text-embedding-004`).
- **Enhanced**: Cascading Model Fallback System (Pro → Flash → Expedited).
- **Protocol**: Fully standardized on **FastMCP v2.0** with SSE transport.

---

## 2. Core Architecture

### 2.1 The "Protocol + Blackboard" State Machine
The system adheres to the "Protocol + Blackboard" paradigm:
- **Blackboard (`project_profile.json`)**: The single source of truth for state, tasks, and context.
- **Protocol (Orchestrator)**: Stateful engine that manages the lifecycle of `Task` objects.
- **Concurrency**: filelock-based access ensures safe concurrent writes between the Orchestrator and parallel Workers.

### 2.2 Model Routing (Smart Cascade)
Requests are routed via `mcp_core/llm.py` using a reliability-first cascade:
1. **Primary**: `gemini-3-flash-preview` (High speed/intelligence balance)
2. **Fallback 1**: `gemini-2.5-flash` (Stable)
3. **Fallback 2**: `gemini-2.5-pro` (Reasoning/Complex tasks)
4. **Local**: `ollama/llama3` (Offline/Private mode - Opt-in)

### 2.3 Retrieval Augmented Generation (RAG)
- **Embeddings**: Switched from local `sentence-transformers` (heavy) to **Gemini API** (`models/text-embedding-004`) for lightweight, high-performance vectorization.
- **HippoRAG**: Maintains knowledge graph capabilities for complex context retrieval, now powered by Gemini's long-context understanding.

## 4. Operational Guide

### 4.1 Configuration
Configuration is managed via environment variables and `project_profile.json`:
- `GEMINI_API_KEY`: Required for all operations.
- `MCP_HOST` / `MCP_PORT`: Server binding (default 0.0.0.0:8000).

### 4.2 Autonomous Git Workflow
The Orchestrator now includes an Autonomous Git Worker capable of:
1. **Detecting Changes**: Monitoring file edits.
2. **Committing**: Generating semantic commit messages via Gemini-3-Flash.
3. **Pull Requests**: Drafting PR titles/bodies and interfacing with GitHub (if configured).

> **Note**: To enable autonomous commits, Ensure your MCP Client connects via SSE (`http://localhost:8000/sse`) to avoid `stdin`/EOF issues.
